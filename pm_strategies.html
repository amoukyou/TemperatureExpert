A类：问题技巧 — 需要领域知识

这类策略的核心是对温度的预判能力。你需要比市场更早、更准确地判断温度走势。换一个话题（政治、体育），这些策略就不适用了——你需要的是气象学知识，不是金融知识。

需要领域知识 风险中等~高
A1. 气象预报套利
对比专业预报与市场定价的信息差
中风险 需气象数据 ▶
核心逻辑
专业气象模型（NOAA GFS、ECMWF）每 6-12 小时更新一次预报，精度远高于人群直觉。
当模型给出的温度概率分布与 Polymarket 市场定价存在显著偏差时，按模型下注即可获利。

本质：信息延迟套利 — 气象数据更新快，市场反应慢，中间存在时间窗口。
操作流程
获取最新气象预报
→
计算各温度区间概率
→
对比 PM 市场价
→
发现偏差 > 阈值
→
买入低估选项
→
持有到结算
举例
场景
GFS 模型预测明天 NYC 最高温 18°C，概率 65%。
但 PM 上 "17-18°C" 区间的 Yes 价格只有 $0.35（隐含概率 35%）。
偏差 = 65% - 35% = 30个百分点 → 买入。

买 100 份 Yes @$0.35 = 花 $35 → 如果中了收 $100 → 赚 $65 (186% ROI)
数据源
  • NOAA GFS：每 6 小时更新（00/06/12/18 UTC），免费 API
  • ECMWF：每 12 小时更新，精度最高，部分数据付费
  • Weather Underground：PM 官方结算数据源
  • Open-Meteo：免费天气 API，汇聚多个模型
识别特征（在我们的数据中）
如何用看板找到这类人
信念比 ≥ 70%（持有到结算）· 均买价 0.15-0.60 · 每事件 1-2 个选项（精准押注）· 稳定盈利
过滤器设置：信念比≥70, 每事件最多选项≤2, 均买价≤0.6, A类盈亏≥0
A2. 温度阶梯策略 (Laddering)
多区间低价广覆盖，中一个回本全部
中风险 ▶
核心逻辑
不赌精确温度，而是在预报范围附近的多个相邻区间同时低价挂单。
比如预报 15°C ± 3°C，就在 12-18°C 的 7 个区间各买一点。

每份价格 $0.02-$0.15，只要中一个就赚 $0.85-$0.98。
以小博大：总投入 $0.50，中一个赚 $0.50-$0.85。
举例
neobrother 的做法（代表人物）
在 Buenos Aires / NYC 温度市场，同时买入 5-7 个相邻区间的 Yes：
14°C @$0.05, 15°C @$0.12, 16°C @$0.18, 17°C @$0.15, 18°C @$0.08, 19°C @$0.03
总投入 = $0.61

如果实际温度 = 16°C → 16°C 那份值 $1，其余归零
赚 $1 - $0.61 = $0.39 (64% ROI)，即使多花了钱买了没中的区间
识别特征
如何用看板找到这类人
每事件多选项（3-7个）· 低均买价（$0.05-$0.15）· 胜率看起来低（因为每个事件只中1个但买了多个）
过滤器设置：每事件最多选项≥3, 均买价≤0.2
A3. 不对称重注 (Asymmetric Big Bet)
大额押注低概率事件，博取超高回报
高风险 ▶
核心逻辑
在市场认为概率很低（5-15%）但你认为被低估的选项上下大注。
如果中了，回报率 700-2000%。如果没中，亏损全部本金。

这种策略依赖于对极端天气事件的判断力 — 比如突然的热浪、寒潮。
经典案例
Hans323 — $92K → $1.11M
押注 London 某天温度会异常高（市场隐含概率 8%）。
投入 $92,632 买入 Yes @$0.08。
实际温度果然创新高 → Yes 结算为 $1。
$92,632 × (1/0.08) ≈ $1,157,900 → 净赚 $1,065,268 (1149% ROI)
识别特征
如何用看板找到这类人
单事件盈利占比极高（>80%）· 参与事件少 · 大额投入 · 盈亏波动剧烈
注意：这类人在我们的"专家过滤"中会被"最大单事件盈利占比≤50%"过滤掉，因为他们靠的是运气+胆量而非持续能力
A4. 气象模型更新窗口 (Latency Arb)
在模型更新后的1-2分钟内抢先下单
低风险 需要 Bot ▶
核心逻辑
GFS 模型每 6 小时更新（00/06/12/18 UTC），ECMWF 每 12 小时。
新预报发布后 1-5 分钟内，多数交易者还未反应，市场价还是旧的。

Bot 自动解析新预报 → 计算新概率 → 对比市场价 → 下单
这是 A1 的自动化高速版本，纯粹比速度。
技术要求
  • 实时订阅气象数据 API（解析 GRIB2 格式）
  • 概率分布计算引擎（将点预报转为区间概率）
  • Polymarket CLOB API 集成（毫秒级下单）
  • 24/7 运行的服务器
识别特征
如何用看板找到这类人
极高交易频率 · 交易时间集中在模型更新后 · 持续小额盈利 · 7天活跃度高
名字通常带 "bot"、"auto" 等关键词

B类：PM 平台通用技巧 — 与问题无关

这类策略不在乎问题本身在问什么，只利用 Yes/No 选项之间的数学关系和市场结构做风险套利。温度、政治、体育、加密货币——任何 Polymarket 市场都能用。交易者玩的不是"答案"，而是"赔率结构"。

通用适用 多为低风险
B1. 扫尾盘等判决
结果已明朗但未结算，抢先买入确定的赢家
极低风险 通用 ▶
核心逻辑
适用于任何结果已经明朗但市场还没正式结算的场景。
温度市场：下午最高温已确定，但市场到晚上才结算。
体育市场：比赛已结束，但结算需要几小时。
政治市场：选举结果已出，但市场还在交易。

在结果已知的窗口期，以 $0.90-$0.98 买入赢家 → 等结算收 $1
不需要预判任何事情，只需要看到"答案已经出来了"。
操作流程
事件结果已出
→
市场尚未结算
→
确认赢家选项
→
买入 Yes @$0.92-0.98
→
结算收 $1.00
买 1000 份 @$0.95 = 花 $950 → 结算收 $1000 → 赚 $50 (5.3%)
为什么是 B 类而非 A 类
虽然温度市场中"扫尾盘"需要看温度数据，但这个策略的本质不是"预测温度"而是"确认已知结果并抢在结算前行动"。
同样的手法在任何 PM 市场上都能用——只要结果已公开但还没结算。
玩家不需要气象学知识，只需要信息获取速度和执行速度。
风险点
  • 数据源不一致：你看到的结果和 PM 官方结算数据源可能不同
  • 价格已被其他 bot 推到 $0.99+，利润极薄
  • 偶尔有争议导致结算延迟或结果反转
识别特征
如何用看板找到这类人
均买价 ≥ $0.85 · 极高胜率 · "总盈亏"高但"A类盈亏"低或为负
在看板中被"近似结算"列捕获
B2. 总价 >$1 套利 (Overround)
所有选项价格之和超过 $1 时，SPLIT 卖出锁定利润
无风险 纯数学 ▶
核心逻辑
温度市场每个事件有 7-11 个互斥选项（温度区间），只有一个会中奖。
理论上所有选项的 Yes 价格之和应该 = $1.00。

当总和 > $1 时：
  1. 花 $1 执行 SPLIT — 铸造每个选项各 1 份 Yes 代币
  2. 按市场价卖出所有选项的 Yes
  3. 总收入 > $1 → 差额就是无风险利润
举例
London 某天温度市场
7 个选项的 Yes 价格：$0.05 + $0.12 + $0.18 + $0.25 + $0.22 + $0.13 + $0.08 = $1.03
SPLIT $1000 → 铸造 1000 套 → 卖出全部 = 收 $1030 → 净赚 $30 (3%)
无论哪个温度中奖，你都已经卖出了所有代币，不持有任何风险。
检测规则（严格版）
必须同时满足：
1 在5分钟内卖出 ≥70% 选项的 Yes（SPLIT 后快速卖出）
2 各选项卖出量大致均匀（最大/最小不超过 3 倍）

做市商分批在不同时间段挂卖单不算 B2。
注意事项
  • 需要逐个选项挂卖单，可能面临滑点
  • 手续费会吃掉部分利润（PM 手续费约 1-2%）
  • 高频 Bot 竞争激烈，套利窗口通常只有几秒
  • 需要足够的资金同时操作多个选项
B3. 总价 <$1 套利 (Underround)
所有选项价格之和不到 $1 时，全买等结算
无风险 纯数学 ▶
核心逻辑
当所有选项的 Yes 价格之和 < $1 时：
  1. 买入每个选项各 1 份 Yes
  2. 总花费 < $1
  3. 结算时必有 1 个中奖 = $1
  4. 差额 = 无风险利润
总花费 $0.94 → 结算必收 $1.00 → 赚 $0.06/套 (6.4% ROI)
检测规则（严格版）
必须同时满足：
1 在5分钟内买入 ≥70% 选项的 Yes（锁定价格窗口）
2 各选项买入量大致均匀（最大/最小不超过 3 倍）

如果分隔数小时或数天买入不同选项，更可能是 A2 温度阶梯或套保策略，不算 B3。
与 B2 的对比
B2 (>$1)B3 (<$1)
操作SPLIT + 卖出全部买入 + 等结算
资金占用短（卖出即回收）长（需等到结算）
频率较常见（市场偏向过度定价）较少见
时间约束5分钟内完成卖出5分钟内完成买入
仓位约束各选项卖出量大致均匀各选项买入量大致均匀
B4. Yes+No 双买套利
同一选项的 Yes+No 价格之和 <$1 时双买
无风险 纯数学 ▶
核心逻辑
Polymarket 每个选项都有 Yes 和 No 两个代币，理论上 Yes价 + No价 = $1.00。

如果 Yes = $0.40, No = $0.55, 总价 = $0.95 < $1：
同时买 Yes 和 No，花 $0.95，结算必收 $1 → 赚 $0.05。

反之如果总价 > $1（如 Yes=$0.45 + No=$0.58 = $1.03）：
SPLIT $1 铸造 Yes+No → 分别卖出 → 收 $1.03 → 赚 $0.03。
检测规则（严格版）
必须同时满足：
1 在5分钟内同时买入同一 conditionId 的 Yes 和 No
2 Yes 和 No 的买入金额大致相当（比例在 0.5x-2x 之间）

如果间隔数小时先买 Yes、后买 No，更可能是对冲或平仓操作。
B5. SPLIT 做市策略
铸造代币后双边挂单，赚取买卖价差
中风险 纯数学 ▶
核心逻辑
做市商不赌方向，靠买卖价差（spread）赚钱。

  1. SPLIT $1 → 铸造 1 份 Yes + 1 份 No
  2. 在 Yes 侧挂卖单 @$0.42（ask）
  3. 在 No 侧挂卖单 @$0.60（ask）
  4. 如果两边都成交 → 收 $0.42 + $0.60 = $1.02 → 赚 $0.02

也可以只卖一边（卖出不看好的），保留另一边当投机仓位。
风险
  • 单边成交风险：只有一边卖出去，另一边被套
  • 价格波动：气象预报更新导致价格剧烈波动，挂单被扫
  • 库存风险：持有大量单边代币，结算时可能归零
识别特征
如何用看板找到这类人
spent = $0（没有买入记录，全是 SELL）· ROI 显示 999% · 高交易次数
这类人的代币来自链上 SPLIT 操作，不会出现在交易 API 中
B6. 子集/逻辑矛盾套利
利用选项之间的逻辑包含关系
无风险 纯数学 ▶
核心逻辑
温度市场的选项有逻辑包含关系：
"≤60°F" 包含了 "56-57°F"、"58-59°F"、"60°F"

如果 "≤60°F" 的 Yes 价格 < "58-59°F" 的 Yes 价格，就存在逻辑矛盾：
子集的概率不可能大于超集。

操作：买入 "≤60°F" Yes + 卖出 "58-59°F" Yes → 无论结果都有利可图。
温度市场的特殊性
  • PM 温度市场的首尾选项是开放区间："≤X°F" 和 "≥Y°F"
  • 这些开放区间是多个具体区间的超集
  • 当市场流动性低时，容易出现定价矛盾
  • 套利窗口通常很短，需要 bot 监控
B7. 未知 B 类技巧 (Unknown Pattern)
极短时间内高频买卖或同时买入多个选项，疑似未知的结构性套利
风险未知 通用 ▶
核心逻辑
这类行为不符合任何已知的 B1-B6 模式，但明显不是"对温度做判断"的 A 类行为。

特征表现为：
  • 在极短时间内（同一分钟或几分钟内）对同一事件进行多次买卖
  • 在同一事件中买入多个不相邻的选项，但不构成完整的阶梯覆盖
  • 快速买入后又快速卖出，像是在刷单、测试流动性或执行某种算法策略

可能的解释：
  • 未被公开的新型数学套利手法
  • 做市 Bot 的报价调整行为（挂单→撤单→重挂）
  • MEV 或抢先交易（front-running）策略
  • 流动性探测（probe 订单簿深度）
  • 跨选项的动态对冲（Delta hedging）
检测规则
同时满足以下条件之一：

规则1：同一事件中，在 5 分钟内有 ≥5 笔交易（高频短线）
规则2：同一事件中，买入了 ≥3 个不同选项的 Yes，且不满足 B3（<$1 套利）的条件

排除已被标记为 B1-B6 的行为后，剩余符合上述规则的标记为 B7。

策略多选项买入？高频？持有到结算？

C类：跨平台技巧 — 利用平台间价差

利用不同预测市场平台（Polymarket、Kalshi、Metaculus 等）对同一事件的定价差异，两边对冲锁定利润。需要多平台账户和资金。

跨平台 低风险
C1. 跨平台对冲套利
同一事件在不同平台的价差
低风险 ▶
核心逻辑
Polymarket 和 Kalshi 都有温度预测市场。同一事件在两个平台上的定价可能不同。

操作：在价格低的平台买 Yes，在价格高的平台买 No（或反之）。
两边对冲，无论结果如何都锁定价差利润。
举例
NYC 温度 18°C
Polymarket: Yes @$0.45
Kalshi: No @$0.52（等于隐含 Yes @$0.48）

在 PM 买 Yes @$0.45 + 在 Kalshi 买 No @$0.52 = 总花费 $0.97
无论结果：一边收 $1 → 赚 $0.03/套 (3.1%)
操作流程
监控多平台价格
→
发现 PM Yes + Kalshi No < $1
→
PM 买 Yes
→
Kalshi 买 No
→
锁定利润
限制与风险
  • 需要在两个平台都有账户和资金
  • Kalshi 仅限美国用户，PM 的 .com 版本不支持美国
  • 结算规则可能不完全一致（数据源、判定标准、结算时间）
  • 资金效率低（两边都要锁定保证金，年化收益可能不高）
  • 汇率和提取手续费侵蚀利润
识别特征
在我们的数据中
这类策略的操作一半在 PM、一半在 Kalshi，我们只能看到 PM 侧的交易。
特征：可能看到只有 Yes 买入（PM侧），没有对应的卖出或结算——因为对冲端在 Kalshi。
